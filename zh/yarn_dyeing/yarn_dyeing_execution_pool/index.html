    Yarn Dyeing Execution Pool 使用说明¶

    中文名称：染纱执行池 / 染纱执行任务池

    1. 这个表单是做什么的？¶

    Yarn Dyeing Execution Pool 是染纱执行任务池。

    它不是最早的染纱需求表，也不是正式生产用的 Work Card。它的作用是把已经确认的 Yarn Dyeing Order 变成可以执行、可以安排、可以跟进状态的染纱任务。

    简单理解：

    表单 员工理解
    Yarn Dyeing Requirement Pool 染纱需求
    Yarn Dyeing Order 染纱订单
    Yarn Dyeing Execution Pool 染纱执行任务
    Yarn Dyeing Work Card 实际染纱生产卡

    染厂负责人或生产相关人员主要用这个表单确认：

    • 这批纱是否已经安排染色；
    • 是哪一种纱、哪个颜色、哪个订单来源；
    • 要安排多少 kg；
    • 计划什么时候完成；
    • 当前 Execution Status 是什么；
    • 是否还有 Balance Qty (kg) 需要继续处理。

    2. 表单流程关系¶

    标准流程是：

    1. Yarn Dyeing Requirement Pool 产生染纱需求。
    2. 染纱需求进入 Yarn Dyeing Order。
    3. Yarn Dyeing Order 确认后，系统生成 Yarn Dyeing Execution Pool。
    4. 染厂负责人从 Yarn Dyeing Execution Pool 安排 Yarn Dyeing Work Card。
    5. Yarn Dyeing Work Card 选择 Execution 后，Execution Status 变成 Scheduled。
    6. Yarn Dyeing Work Card 完成后，Execution Status 变成 Completed。

    Yarn Dyeing Execution Pool 是 Yarn Dyeing Work Card 的来源。没有对应 Execution 时，不要随便直接建立 Work Card。

    注意：Output Record 是产出记录，不是完成按钮。不要以为填写了日报，Yarn Dyeing Execution Pool 就会自动完成。Execution Status 的 Scheduled / Completed 主要跟 Yarn Dyeing Work Card 的安排和完成有关。

    3. 打开表单后先看什么？¶

    第一步：先看状态¶

    先看 Execution Status。

    Execution Status 含义
    Pending 还没有安排到 Yarn Dyeing Work Card。
    Scheduled 已经安排到 Yarn Dyeing Work Card，但不代表已经完成。
    Completed 这条染纱执行任务已经完成。

    注意：Scheduled 只是已经安排，Completed 才是完成。

    第二步：再看来源¶

    重点查看这些来源字段：

    • Yarn Dyeing Execution No.
    • Yarn Dyeing Order No.
    • Yarn Dyeing Requirement No.
    • Elastic Source WO No.
    • Elastic Source Order No.
    • Elastic Code
    • Job Type
    • Source Type
    • Requirement Type

    这些字段帮助你判断这条任务来自哪个订单、哪个 Elastic 工单、是样板还是大货、是内部 Elastic 需求还是外部染纱服务。

    第三步：确认纱线和颜色¶

    重点查看：

    • Yarn Item Code
    • Standard Item Name
    • Brand
    • Target Color Ref.
    • Color Name
    • Dyeing Process
    • Light Source
    • Yarn Color Standard No.
    • Special Requirements

    不要只看 Color Name。如果有 Target Color Ref.，也要一起核对。

    第四步：确认数量¶

    重点查看：

    字段 含义
    BOM Calculated Qty (kg) BOM 计算出来的需求数量。
    Yarn Dyeing Order Qty (kg) Yarn Dyeing Order 中安排的总染纱数量。
    Execution Planned Qty (kg) 本次执行计划染多少 kg。
    Balance Qty (kg) 剩余数量。

    Balance Qty (kg) 通常按 Yarn Dyeing Order Qty (kg) 减 Execution Planned Qty (kg) 计算。它是系统计算字段，不要手工修改。

    第五步：确认日期和优先级¶

    重点查看：

    • Planned Start Date
    • Planned Finish Date
    • Priority
    • Complete Time

    这些字段帮助染厂负责人安排先后顺序，并跟进实际完成情况。

    4. 关于智能助手 / 自动化¶

    这个表单有多处自动化，不需要员工手工重复做。

    自动化 1：确认 Yarn Dyeing Order 后生成 Execution¶

    Yarn Dyeing Order 确认后，系统生成 Yarn Dyeing Execution Pool。

    系统会带入相关来源、纱线、颜色、工艺和数量资料。员工不要为了快而随便手工新增 Execution 记录。

    自动化 2：Work Card 安排后更新状态¶

    Yarn Dyeing Work Card 选择 Execution 后，Execution Status 变成 Scheduled。

    自动化 3：Work Card 完成后更新状态¶

    Yarn Dyeing Work Card 完成后，Execution Status 变成 Completed，并记录 Complete Time。

    自动化 4：Balance 继续染时生成后续需求¶

    当 Balance Qty (kg) > 0，并且确实需要继续染剩余数量时，才选择 Create Balance Requirement? = Yes。

    系统会生成后续或补染需求，并把 Balance Requirement Created? 设为 Yes，用于防止重复生成。

    注意：Balance Qty (kg) > 0 不代表一定要选择 Create Balance Requirement? = Yes。只有剩余数量确实还要继续染时才选择。

    5. Requirement Type 怎么理解？¶

    Requirement Type 表示这条染纱执行任务的需求类型。

    常见值：

    Requirement Type 员工理解
    New Order 新订单或第一次染纱需求。
    Replenishment 剩余数量或追加数量再次安排。
    Rework 因问题需要重新染或返工。

    Requirement Type 通常由来源表单或系统带出，不要随便修改。

    6. 哪些字段可以改？¶

    染厂负责人可以根据实际安排修改：

    • Planned Start Date
    • Planned Finish Date
    • Priority
    • Execution Planned Qty (kg)
    • Create Balance Requirement?
    • Complete Time
    • Special Requirements / Remarks

    修改前要确认真实生产安排。尤其是 Execution Planned Qty (kg) 和 Create Balance Requirement?，会影响后续数量和 Balance 处理。

    Execution Status 一般应由 Yarn Dyeing Work Card 自动回写，不建议员工手工修改。

    7. 哪些字段不要随便改？¶

    以下字段不要随便改：

    • Yarn Dyeing Execution No.
    • Yarn Dyeing Order No.
    • Yarn Dyeing Requirement No.
    • Elastic Source WO No.
    • Elastic Source Order No.
    • Elastic Code
    • Job Type
    • Source Type
    • Requirement Type
    • Yarn Item Code
    • Standard Item Name
    • Target Color Ref.
    • Color Name
    • Dyeing Process
    • Yarn Dyeing Order Qty (kg)
    • Balance Qty (kg)
    • Balance Requirement Created?

    原因很简单：这些字段大多来自上游表单、产品资料、颜色资料、系统计算或自动化控制。乱改会导致 Work Card、订单来源、数量和后续补染关系不一致。

    8. 不能用 Manage Data 后台批量修改的重要字段¶

    不要在 Manage Data 后台批量修改：

    • Execution Status
    • Create Balance Requirement?
    • Balance Requirement Created?
    • Execution Planned Qty (kg)

    后台批量修改可能不会触发智能助手 / 自动化。

    正确做法：

    1. 打开单条 Yarn Dyeing Execution Pool 记录。
    2. 按正常页面修改需要修改的字段。
    3. 保存记录。
    4. 让系统按正常流程触发智能助手 / 自动化。

    如果需要批量处理很多记录，先和负责人确认处理方法，不要直接在 Manage Data 里批量改关键字段。

    9. 常见错误¶

    常见错误 正确做法
    看到 Scheduled 就以为已经完成。 Scheduled 只是已经安排，Completed 才是完成。
    看到 Balance Qty (kg) > 0 就直接选择 Create Balance Requirement? = Yes。 只有剩余数量确实还要继续染时才选择。
    手工修改 Balance Requirement Created?。 这是系统防重复字段，不要手工修改。
    在 Execution Pool 里改来源订单、纱线或颜色。 这些资料应来自上游需求、订单或产品资料；发现错误时先检查来源表单。
    用 Manage Data 批量改关键字段。 打开单条记录正常修改并保存。

    10. 员工记忆口诀¶

    先看状态，再看来源，再看纱线，再看颜色，再看数量，最后看余额。

    最重要的一句话：

    Yarn Dyeing Execution Pool 是用来安排 Yarn Dyeing Work Card、跟进 Execution Status、控制执行数量和 Balance 的染纱执行任务池，不是重新填写染纱需求的地方。
