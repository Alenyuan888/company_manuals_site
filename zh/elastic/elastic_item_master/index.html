    Elastic Item Master 使用说明¶

    中文名称：松紧带产品主数据 / Elastic 产品资料库

    1. 表单用途¶

    Elastic Item Master 是 Elastic 产品主数据表。

    当前确认规则：一条记录 = 一个客户下面的一款 Elastic 产品。 不要在一条记录中保存多款产品。

    本表保存：

    • 客户、业务归属和产品基本规格；
    • Original Sample（OS，客户原样）的分析和 BOM；
    • Final Base Sample（FBS，最终确认样）的规格和 BOM。

    2. 谁使用¶

    • Designer Department：填写或检查产品规格、OS 分析和 BOM。
    • Sales Department：确认客户、Responsible Sales、Sales Account 和产品资料。
    • Jodoo Admin：维护权限、字段、公式和智能助手。

    3. 三个页面¶

    1. Elastic Specification：客户、业务归属和产品基本规格。
    2. Elastic Sample Analysis：OS 分析、技术参数和 OS BOM。
    3. Elastic FBS Record：最终确认技术标准和 FBS BOM。

    OS 和 FBS 必须分开保存，不要互相覆盖。

    4. 新建记录步骤¶

    1. 先按客户和产品搜索，避免重复建档。
    2. 在 Customer 使用 Select Data 从 Elastic Customer 选择客户。
    3. 检查 Customer Code、Company Name 和 Country。
    4. 填写 Responsible Sales，并选择正确的 Sales Account。
    5. 填写 Elastic Type、Description、Elastic Color、Production Location、Width、Width Unit 和 Net Weight (g/m)。
    6. 按实际工作阶段填写 OS 或 FBS 页面。
    7. 每种 BOM 材料填写一行，并检查单位、根数和单耗。
    8. 保存前再次核对客户、Sales Account、规格和 BOM。

    5. Sales Account 为什么重要¶

    Sales Account 是 Member 字段，用于记录归属、Publish 权限和可见范围。

    • Responsible Sales 可以是业务名称或选择字段。
    • 真正控制哪个账号可以看到记录的是 Sales Account。
    • Bangladesh Sales Team 可以共用一个 Sales Account；中国销售可以选择个人 Sales Account。

    Sales Account 为空或选择错误，可能导致记录归属错误或业务人员看不到资料。

    6. Elastic Specification 当前字段¶

    • Customer；Customer Code；Company Name；Country
    • Responsible Sales；Sales Account；Sample File
    • Elastic Type；Description；Elastic Color；Production Location
    • Width；Width Unit；Net Weight (g/m)
    • Elastic Code；Quality Requirements

    7. OS 与 FBS¶

    OS 页面记录客户原样分析，不代表最终生产标准。FBS 页面保存最终确认标准，未经确认的资料不要作为 FBS 填写。

    8. OS BOM 与 FBS BOM¶

    两个 BOM 子表当前确认的字段顺序相同：

    顺序 字段 填写说明
    1 Select Data 从 Item Master Data 选择材料
    2–9 Position；Item Code；Item Category；Material Type；Standard Item Name；材料中文名；Base Unit；Brand 选择或由关联资料带出；精确映射见待核对项
    10 No. of Ends 填写材料使用根数
    11 Unit Consumption (g/m) 填写每米单耗，不是总需求量
    12–17 Need Dyeing?；Dyeing Process；Color Name；Target Color Ref.；Quality Reference Type；Quality Reference No. 按实际染色和质量要求填写
    18–19 Remarks；Confirm Status 补充说明并按当前流程确认

    最后一个系统字段不同：OS 使用 单行BOM信息打包，FBS 使用 FBS单行BOM信息打包。当前 OS/FBS BOM 均没有 Light Source。

    9. 与 Elastic Update by Designer 的关系¶

    Elastic Update by Designer 用于收集或补充产品资料，不直接修改已有 Elastic Item Master。

    当前已确认：Sales Confirmation = Confirm 且 Sales Account 不为空时，已启用的智能助手新增一条 Elastic Item Master 记录，并把一份 Common BOM 分别复制到 OS BOM 和 FBS BOM。Reject 不同步。

    10. 不要手工修改的字段¶

    • Elastic Code
    • 全局BOM字符串集合
    • FBS全局BOM字符串集合
    • 单行BOM信息打包
    • FBS单行BOM信息打包

    这些字段用于编号、BOM 打包、比较和自动化。

    11. 常见错误¶

    • 同一客户和产品重复建档。
    • 一条记录填写多款 Elastic 产品。
    • Sales Account 为空或选错。
    • 把 OS 资料当成最终 FBS。
    • BOM 材料、单位、根数或单耗错误。
    • 手工修改系统生成字段。
    • 误以为 Elastic Update by Designer 会直接修改已有记录。
