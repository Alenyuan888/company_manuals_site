    Elastic Order Request 使用说明¶

    中文名称：松紧带销售订单 / 松紧带订单申请表

    1. 表单用途¶

    Elastic Order Request 是松紧带业务的销售订单和样板申请入口表单。

    销售人员在本表录入：

    • 客户和订单基础信息；
    • Job Type、Billing Type、客户 PO 和预计交期；
    • Elastic Order Details 中的产品明细、数量、颜色、宽度和生产路线；
    • 附件和备注。

    生产部门从安排生产角度查看本表，重点关注审批结果、预计交期、产品明细和 Production Arrangement Status，判断订单是否待安排、已安排、已完成或取消。

    本表也是后续同步 Elastic Sales Order 辅助表、以及在 Sample 批准后生成 Elastic Sample Development Work Order 的来源。

    2. 谁使用、什么时候使用¶

    主要使用部门：

    • Sales
    • Production
    • Management

    没有单独的 Design Department。涉及设计师时，统一理解为生产/设计人员，使用 Production 同一账号和权限体系。

    常见使用场景：

    1. 销售收到客户样板需求或大货订单。
    2. 销售需要录入客户、订单、产品明细、数量、交期和附件。
    3. 主管需要在 workflow 中审批订单。
    4. Job Type = Sample 且主管批准后，系统生成样板开发工单。
    5. 生产部门按 Production Arrangement Status 跟踪订单是否需要安排生产。

    3. 当前 workflow¶

    当前 workflow：

    1. Enter Order Details
    2. Supervisor Approval
    3. End

    Supervisor Approval 是 Regular Approval 节点。员工只需要知道：订单资料填好后提交主管审批；审批通过后，后续自动化和生产安排才可以继续处理。

    4. Production Arrangement Status¶

    Production Arrangement Status 用于生产部门从排产或安排生产角度判断订单状态。

    选项 含义
    Pending 订单已存在，但生产尚未安排或尚未确认。
    Scheduled 生产已安排，或已进入后续样板/大货处理。
    Completed 生产相关处理已完成。
    Cancelled 订单取消或不再继续处理。

    生产部门日常优先查看 Approval Result = Approve 且 Production Arrangement Status = Pending 的记录。

    5. 销售填写步骤¶

    1. 打开 Elastic Order Request。
    2. 在 Select Customer 选择正确客户。
    3. 核对 Customer Code、Company Name、Country、Responsible Sales 和 Sales Account。
    4. 正确选择 Job Type：例如 Sample 或 Bulk，以系统当前选项为准。
    5. 正确选择 Billing Type：例如 FOC 或 CHG，以系统当前选项为准。
    6. 填写 Customer PO No.、Purchase Order Date 和 Expected Delivery Date。
    7. 在 Attachment 上传客户订单、图片、规格图或参考资料。
    8. 在 Elastic Order Details 中逐行填写产品明细。
    9. 不同款式、颜色、宽度或客户要求的松紧带，应分开成不同明细行。
    10. 认真选择 Production Routing，它会影响后续样板或大货处理。
    11. 提交给主管审批。

    6. Sales Order Information¶

    这些字段用于识别客户、订单来源、业务归属和生产安排状态。

    字段 员工填写说明
    Select Customer 选择正确客户资料。
    Customer Code 客户编号；用于后续同步和查找。
    Company Name 客户公司名称；选择客户后请核对。
    Country 客户国家或地区。
    BIN 客户 Business Identification Number，如适用。
    Responsible Sales 负责业务员。
    Sales Account 数据归属和权限相关字段；必须选择正确。
    Job Type 区分 Sample / Bulk 等工作类型；按系统当前选项选择。
    Billing Type 区分 FOC / CHG 等计费类型；按系统当前选项选择。
    Customer PO No. 客户 PO 或参考单号。
    Purchase Order Date 客户订单日期。
    Attachment 客户订单、图片、规格图或参考资料。
    Expected Delivery Date 客户期望交期，生产会重点查看。
    Remarks 订单备注。
    Elastic Order No. 系统订单编号或内部订单号；用于后续查找和同步。
    Production Arrangement Status 生产安排状态。

    7. Elastic Order Details¶

    Elastic Order Details 是订单产品明细子表。一行代表一个松紧带款式或产品明细。

    字段 员工填写说明
    Select Elastic 选择对应 Elastic 产品资料。
    Elastic Code Elastic 产品编号；用于后续同步和查找。
    Elastic Type Elastic 类型，按系统资料或当前选项确认。
    Description 产品描述。
    Elastic Color 颜色。
    Target Color Ref. 目标色号或参考色。
    Order Ref. 客户订单中的产品参考。
    Width / Width Unit 宽度和单位。
    Net Weight 净重，如适用。
    Quantity / Quantity Unit 数量和单位。
    Production Routing 生产路线；生产部门会重点查看。
    Remarks 产品明细备注。
    Product Picture 产品图片或参考图。
    Billing Type(辅助) 系统辅助字段，等于主表 Billing Type。
    Work Type(辅助) 系统辅助字段，等于主表 Job Type。

    销售商业/结算相关字段应按公司权限和流程填写。生产部门如果在自己的视图中看到不属于生产处理的商业/结算字段，不应修改，需按权限问题反馈给 Admin 或 Alen。

    8. 生产部门如何快速筛选订单¶

    如果生产部门不知道 Elastic Order No.，不要只靠单号查找。建议按以下顺序筛选：

    1. 筛 Responsible Sales。
    2. 筛 Job Type = Sample / Bulk。
    3. 筛 Customer Code 或 Company Name。
    4. 筛 Production Arrangement Status。
    5. 筛 Expected Delivery Date。
    6. 再用 Elastic Code、Order Ref.、Width、Elastic Color 找到具体产品明细。

    生产部门最需要看的字段：

    • Approval Result
    • Production Arrangement Status
    • Job Type
    • Expected Delivery Date
    • Responsible Sales
    • Customer Code
    • Company Name
    • Elastic Order No.
    • Customer PO No.
    • Elastic Code
    • Elastic Color
    • Width
    • Quantity
    • Production Routing

    建议生产视图显示字段包括 Approval Result、Production Arrangement Status、Job Type、Expected Delivery Date、Responsible Sales、Customer Code、Company Name、Elastic Order No.、Customer PO No.、Submission Time 或 Purchase Order Date。这是建议配置，不代表当前系统已经完成配置。

    9. Review & Approval¶

    审批相关字段用于主管判断订单是否可以继续处理。

    字段 说明
    Risk Assessment 风险评估。
    PO Status PO 状态。
    Funding Status 资金状态。
    L/C Readiness L/C 准备情况。
    Priority 优先级。
    Remarks 审批备注。
    Order Approval 订单审批字段。
    Approval Result 审批结果。
    Approval Time 审批时间。
    Approver 审批人。
    Reason for Rejection 拒绝原因。

    如被拒绝，应查看 Reason for Rejection 并按主管意见修改后再处理。

    10. Production Status¶

    生产状态字段：

    • Sample Status
    • Bulk Production Status

    这两个字段用于跟踪样板和大货的后续处理状态。具体状态选项以系统当前显示为准。

    11. 保存或审批后会发生什么¶

    已确认的自动化关系：

    1. 松紧带销售订单更新到辅助表 会把 Elastic Order Request 数据同步到 Elastic Sales Order 辅助表。
    2. Job Type = Sample 且 Supervisor Approval 通过时，松紧带生产申请 批准后 生成松紧带样板Work Card 会生成 Elastic Sample Development Work Order。

    员工注意：

    • Customer Code、Elastic Order No. 和 Elastic Code 会影响同步和匹配。
    • Job Type 选错，可能导致样板工单不生成或后续流程错误。
    • Production Routing 选错，会影响生产/样板处理。

    12. 常见错误¶

    • Select Customer 选错，导致客户信息错误。
    • Job Type 把 Sample 和 Bulk 选错。
    • Billing Type 选错。
    • Expected Delivery Date 没填或不准确。
    • Elastic Order Details 把不同款式、颜色或宽度混在同一行。
    • Production Routing 没认真选择。
    • 忘记上传客户订单、图片或规格附件。
    • 生产部门只按单号查找，找不到订单；应使用组合筛选。
    • 未审批或未批准的订单被当作已确认生产订单处理。
