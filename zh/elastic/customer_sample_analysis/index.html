    Customer Sample Analysis 使用说明¶

    中文名称：客户样板分析表

    1. 表单用途¶

    Customer Sample Analysis 是生产部门用于分析客户松紧带样板的技术资料表。

    本表用于记录：

    • 客户样板的结构、织法、宽度、厚度、手感、弹力和重量；
    • 样板材料组成和 Sample Composition Breakdown；
    • BOM 材料、根数、单耗、是否需要染色、染色方式和颜色资料；
    • 客户样板图片、分析照片或参考附件。

    填写完成后，系统通过智能助手把分析结果写回已有 Elastic Item Master 的 OS 技术资料和样板材料组成信息。

    本表不是销售订单表，不记录业务结算资料或生产排期。

    2. 谁使用、什么时候使用¶

    主要使用部门：Production Department。

    没有单独的 Design Department。生产/设计人员使用同一账号和权限体系；本文统一写作“生产/设计人员”。

    常见使用场景：

    1. 客户提供实物样板，需要拆解和分析。
    2. 客户提供图片或已有 Elastic 资料，需要补充技术资料。
    3. 需要建立客户样板的材料组成和 OS BOM。
    4. 需要把样板分析结果同步回已有 Elastic Item Master。

    3. 当前流程¶

    • 本表当前没有 workflow。
    • 本表通过智能助手写回已有 Elastic Item Master。
    • 本表只写回已有记录，不负责新建 Elastic Item Master。

    4. Basic Order Information¶

    这些字段用于确认本次样板分析对应哪个客户、哪一条 Elastic 产品资料。

    字段 员工填写说明
    Select Elastic 选择正确的已有 Elastic / Elastic Item Master 资料。
    Customer Code 写回 Elastic Item Master 的关键匹配条件；不要乱改。
    Elastic Code 写回 Elastic Item Master 的关键匹配条件；不要乱改。
    Country 客户所在国家或地区；选择后请核对。
    Responsible Sales 负责业务员；用于业务追溯。
    Description 产品描述，按样板或已有资料填写。
    Elastic Type 按系统当前选项选择。
    Sales Account 数据归属和后续可见范围相关字段；选择错误会影响资料归属。

    操作重点：

    1. 先选择正确的 Select Elastic。
    2. 保存前确认 Customer Code 和 Elastic Code 与目标 Elastic Item Master 一致。
    3. 如果 Customer Code 或 Elastic Code 不确定，先停止并询问主管或 Jodoo Admin。

    5. Basic Analysis Information¶

    这些字段用于记录样板本身的技术分析结果。

    字段 员工填写说明
    Analysis No. 分析编号；用于后续追溯。
    Accountable Designer 负责分析的生产/设计人员。
    Attachment 上传客户样板图片、分析照片或参考资料。
    Elastic Category 按系统当前选项选择。
    Structure 根据实际样板结构填写。
    Weaving Method 根据实际织法填写。
    Woven Pattern 梭织图案或相关结构说明。
    Knitted Pattern 针织图案或相关结构说明。
    Elastic Name 样板或产品名称。
    Elastic Letter 样板字母或标识。
    Handfeel 手感描述。
    Greige Width / Greige Width Unit 胚带宽度和单位。
    Finish Width / Finish Width Unit 成品宽度和单位。
    Thickness (mm) 厚度，单位为 mm。
    Weft Density (pick/cm) 纬密，单位为 pick/cm。
    Greige Elongation 胚带伸长率。
    Finish Elongation 成品伸长率。
    Sample Weight (g/m) 样板每米重量。
    Created At 创建时间或分析记录时间。

    填写原则：

    • 根据实际样板分析填写，不确定不要猜。
    • 宽度、厚度、弹力、重量等数据要尽量准确。
    • Attachment 应上传能支持分析结论的图片或资料。

    6. Customer Sample Material Analysis¶

    子表名称：Sample Composition Breakdown。

    一行代表一种材料或一个材料位置，不要把多种材料混在同一行。

    字段 员工填写说明
    Position 材料位置或结构位置。
    Select Material 选择正确材料资料。
    Item Code 必须选择正确；不确定先问主管。
    Item Category 材料分类，按系统资料或带出结果确认。
    Material Type 材料类型。
    Standard Item Name 标准材料名称。
    材料中文名 系统材料中文名，保留原字段名。
    Base Unit 基础单位。
    Brand 品牌。
    No. of Ends 根数。
    Unit Consumption (g/m) 每米单耗；会影响 BOM 和后续用量判断。
    Need Dyeing? 判断该材料是否需要染色。
    Dyeing Process 需要染色时填写染色方式。
    Color Name 颜色名称。
    Target Color Ref. 目标色号或参考色。
    Quality Reference Type 质量参考类型。
    Quality Reference No. 质量参考编号。
    Remarks 备注。
    Confirm Status 标记该材料行是否确认；未确认内容不要当作最终 BOM。
    单行BOM信息打包 系统打包字段，请勿手工修改。

    7. 新建分析记录步骤¶

    1. 打开 Customer Sample Analysis。
    2. 点击 New。
    3. 在 Select Elastic 选择正确的已有 Elastic 资料。
    4. 核对 Customer Code 和 Elastic Code。
    5. 核对 Country、Responsible Sales、Description、Elastic Type 和 Sales Account。
    6. 填写 Analysis No.、Accountable Designer 和 Attachment。
    7. 按样板实际情况填写结构、织法、图案、名称、手感、宽度、厚度、弹力和重量。
    8. 在 Sample Composition Breakdown 中逐行填写材料组成。
    9. 检查 Item Code、No. of Ends、Unit Consumption (g/m)、Need Dyeing?、Dyeing Process、Color Name 和 Target Color Ref.。
    10. 对已确认的材料行设置正确的 Confirm Status。
    11. 保存前再次检查 Customer Code、Elastic Code 和 BOM 信息。

    8. 保存后会发生什么¶

    已确认的智能助手：

    • 智能助手名称：材料分析表填写 写回 总表单
    • 触发表单：Customer Sample Analysis
    • 触发条件：新增记录 Add record
    • 动作：更新已有 Elastic Item Master
    • 匹配条件：
    • Elastic Item Master.Customer Code = Customer Sample Analysis.Customer Code
    • Elastic Item Master.Elastic Code = Customer Sample Analysis.Elastic Code

    重要规则：

    • 如果没有匹配到已有 Elastic Item Master，不要自动新增。
    • Customer Sample Analysis 只负责写回已有 Elastic Item Master，不负责创建新的 Elastic Item Master。
    • 因此 Customer Code 和 Elastic Code 必须准确。

    9. 与 Elastic Item Master 的关系¶

    Customer Sample Analysis 是 Elastic Item Master 中 OS 技术资料和样板材料组成的重要来源。

    当前确认会写回 OS 技术字段，包括但不限于：

    • Ref. Analysis No. = Analysis No.
    • Accountable Designer = Accountable Designer
    • Attachment (OS) = Attachment
    • Elastic Category (OS) = Elastic Category
    • Structure (OS) = Structure
    • Weaving Method (OS) = Weaving Method
    • Woven Pattern (OS) = Woven Pattern
    • Knitted Pattern (OS) = Knitted Pattern
    • Elastic Name (OS) = Elastic Name
    • Handfeel (OS) = Handfeel
    • Greige Width (OS) = Greige Width
    • Greige Width Unit (OS) = Greige Width Unit
    • Finish Width (OS) = Finish Width
    • Finish Width Unit (OS) = Finish Width Unit
    • Thickness (mm) (OS) = Thickness (mm)
    • Weft Density (pick/cm) (OS) = Weft Density (pick/cm)
    • Greige Elongation (OS) = Greige Elongation
    • Finish Elongation (OS) = Finish Elongation
    • Sample Weight (g/m) (OS) = Sample Weight (g/m)
    • Created At (OS) = Created At
    • 全局BOM字符串集合 = 全局BOM字符串集合，如字段存在

    10. 常见错误¶

    • 选择了错误的 Select Elastic。
    • Customer Code 或 Elastic Code 与目标 Elastic Item Master 不一致。
    • 不确定技术数据时猜填。
    • 把多种材料混在一行。
    • Item Code 选错。
    • Unit Consumption (g/m) 填成总用量。
    • Need Dyeing? 判断错误。
    • 需要染色时没有填写 Dyeing Process、Color Name 或 Target Color Ref.。
    • Confirm Status 未确认却当成最终 BOM。
    • 忘记上传 Attachment，后续无法追溯样板来源。

    11. 不属于本表的内容¶

    本表不负责：

    • 销售订单；
    • 业务结算资料；
    • 生产排期；
    • 新建 Elastic Item Master。
