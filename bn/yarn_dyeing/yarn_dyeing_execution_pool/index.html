    Yarn Dyeing Execution Pool ব্যবহার নির্দেশনা¶

    1. এই form কী কাজের জন্য?¶

    Yarn Dyeing Execution Pool হলো yarn dyeing execution task pool।

    এটি প্রথম dyeing requirement form নয়, এবং এটি production Work Card-ও নয়। Yarn Dyeing Order confirm হওয়ার পরে system এই form-এ executable dyeing task তৈরি করে। Dyeing supervisor এখান থেকে task arrange ও follow-up করেন।

    সহজভাবে বুঝুন:

    Form Employee meaning
    Yarn Dyeing Requirement Pool Dyeing requirement
    Yarn Dyeing Order Dyeing order
    Yarn Dyeing Execution Pool Dyeing execution task
    Yarn Dyeing Work Card Actual dyeing production card

    Dyeing supervisor সাধারণত দেখেন:

    • yarn ঠিক আছে কি না;
    • color ঠিক আছে কি না;
    • কত kg arrange করতে হবে;
    • finish date কখন;
    • Execution Status কী;
    • Balance Qty (kg) বাকি আছে কি না।

    2. Form process relationship¶

    সাধারণ process:

    1. Yarn Dyeing Requirement Pool থেকে dyeing requirement তৈরি হয়।
    2. Requirement যায় Yarn Dyeing Order-এ।
    3. Yarn Dyeing Order confirm হলে system Yarn Dyeing Execution Pool তৈরি করে।
    4. Dyeing supervisor Yarn Dyeing Execution Pool থেকে Yarn Dyeing Work Card arrange করেন।
    5. Yarn Dyeing Work Card Execution select করলে Execution Status হয় Scheduled।
    6. Yarn Dyeing Work Card complete হলে Execution Status হয় Completed।

    Yarn Dyeing Execution Pool হলো Yarn Dyeing Work Card-এর source। Related Execution ছাড়া Work Card casually তৈরি করবেন না।

    Note: Output Record dyeing output record করে। এটি Yarn Dyeing Execution Pool complete করার button নয়। daily বা output record পূরণ করলেই Execution automatically complete হবে—এভাবে ধরবেন না। Scheduled / Completed মূলত Yarn Dyeing Work Card arrange এবং complete হওয়ার সাথে related।

    3. Form খুলে আগে কী দেখবেন?¶

    Step 1: আগে status দেখুন¶

    প্রথমে Execution Status দেখুন।

    Execution Status Meaning
    Pending এখনো Yarn Dyeing Work Card-এ arrange হয়নি।
    Scheduled Yarn Dyeing Work Card-এ arrange হয়েছে, কিন্তু complete হয়নি।
    Completed এই dyeing execution task complete হয়েছে।

    মনে রাখুন: Scheduled মানে arrange হয়েছে। Completed মানে কাজ শেষ।

    Step 2: Source দেখুন¶

    এই source fields দেখুন:

    • Yarn Dyeing Execution No.
    • Yarn Dyeing Order No.
    • Yarn Dyeing Requirement No.
    • Elastic Source WO No.
    • Elastic Source Order No.
    • Elastic Code
    • Job Type
    • Source Type
    • Requirement Type

    এই fields দিয়ে বোঝা যায় task কোন order, কোন Elastic WO, sample না bulk, এবং internal Elastic requirement না external dyeing service থেকে এসেছে।

    Step 3: Yarn এবং color দেখুন¶

    এই fields দেখুন:

    • Yarn Item Code
    • Standard Item Name
    • Brand
    • Target Color Ref.
    • Color Name
    • Dyeing Process
    • Light Source
    • Yarn Color Standard No.
    • Special Requirements

    শুধু Color Name দেখবেন না। যদি Target Color Ref. থাকে, সেটিও একসাথে check করুন।

    Step 4: Quantity দেখুন¶

    Field Meaning
    BOM Calculated Qty (kg) BOM থেকে calculated requirement quantity।
    Yarn Dyeing Order Qty (kg) Yarn Dyeing Order-এ arranged total dyeing quantity।
    Execution Planned Qty (kg) এই execution-এ plan করা dyeing quantity।
    Balance Qty (kg) remaining quantity।

    Balance Qty (kg) সাধারণত Yarn Dyeing Order Qty (kg) minus Execution Planned Qty (kg) হিসেবে calculate হয়। এটি system calculated field। manually edit করবেন না।

    Step 5: Date এবং priority দেখুন¶

    এই fields দেখুন:

    • Planned Start Date
    • Planned Finish Date
    • Priority
    • Complete Time

    এই fields দিয়ে supervisor priority arrange এবং actual completion follow-up করেন।

    4. Smart Assistant / automation¶

    এই form-এ কিছু কাজ automatic হয়। সবকিছু manually করার দরকার নেই।

    Automation 1: Yarn Dyeing Order confirm হলে Execution তৈরি হয়¶

    Yarn Dyeing Order confirm হওয়ার পরে system Yarn Dyeing Execution Pool তৈরি করে।

    System source, yarn, color, process এবং quantity information নিয়ে আসে। দ্রুত করার জন্য employees manually Execution record add করবেন না।

    Automation 2: Work Card arrange হলে status update হয়¶

    Yarn Dyeing Work Card Execution select করলে Execution Status হয় Scheduled।

    Automation 3: Work Card complete হলে status update হয়¶

    Yarn Dyeing Work Card complete হলে Execution Status হয় Completed, এবং Complete Time record হয়।

    Automation 4: Balance থাকলে শুধু প্রয়োজন হলে follow-up requirement তৈরি হবে¶

    যখন Balance Qty (kg) > 0 এবং remaining quantity সত্যিই dyeing করতে হবে, তখনই Create Balance Requirement? = Yes select করুন।

    System follow-up বা replenishment requirement তৈরি করে এবং duplicate prevent করার জন্য Balance Requirement Created?-কে Yes করে।

    মনে রাখুন: Balance Qty (kg) > 0 মানেই সবসময় Create Balance Requirement? = Yes নয়। শুধু remaining quantity সত্যিই dyeing করতে হলে select করুন।

    5. Requirement Type কীভাবে বুঝবেন?¶

    Requirement Type dyeing requirement-এর type দেখায়।

    Requirement Type Meaning
    New Order নতুন order বা first dyeing requirement।
    Replenishment remaining বা additional quantity আবার arrange করা।
    Rework problem থাকলে re-dye বা rework।

    Requirement Type সাধারণত source form বা system থেকে আসে। casually change করবেন না।

    6. কোন fields পরিবর্তন করা যায়?¶

    Dyeing supervisor real arrangement অনুযায়ী এই fields update করতে পারেন:

    • Planned Start Date
    • Planned Finish Date
    • Priority
    • Execution Planned Qty (kg)
    • Create Balance Requirement?
    • Complete Time
    • Special Requirements / Remarks

    Edit করার আগে actual production arrangement confirm করুন। বিশেষ করে Execution Planned Qty (kg) এবং Create Balance Requirement? পরে quantity এবং balance handling-কে প্রভাবিত করে।

    Execution Status সাধারণত Yarn Dyeing Work Card থেকে automatically write back হবে। manually change করা উচিত নয়।

    7. কোন fields casually change করবেন না?¶

    এই fields casually edit করবেন না:

    • Yarn Dyeing Execution No.
    • Yarn Dyeing Order No.
    • Yarn Dyeing Requirement No.
    • Elastic Source WO No.
    • Elastic Source Order No.
    • Elastic Code
    • Job Type
    • Source Type
    • Requirement Type
    • Yarn Item Code
    • Standard Item Name
    • Target Color Ref.
    • Color Name
    • Dyeing Process
    • Yarn Dyeing Order Qty (kg)
    • Balance Qty (kg)
    • Balance Requirement Created?

    কারণ এই fields সাধারণত upstream form, item data, color data, system calculation বা automation control থেকে আসে। ভুল edit করলে Work Card, source order, quantity এবং balance relation mismatch হতে পারে।

    8. Manage Data-তে batch edit করা যাবে না এমন important fields¶

    Manage Data backend-এ এই fields batch edit করবেন না:

    • Execution Status
    • Create Balance Requirement?
    • Balance Requirement Created?
    • Execution Planned Qty (kg)

    Backend batch edit করলে Smart Assistant / automation trigger নাও হতে পারে।

    সঠিক পদ্ধতি:

    1. একটি Yarn Dyeing Execution Pool record খুলুন।
    2. normal page-এ প্রয়োজনীয় field edit করুন।
    3. Save করুন।
    4. normal process দিয়ে system-কে Smart Assistant / automation trigger করতে দিন।

    অনেক record handle করতে হলে আগে responsible person-এর সাথে method confirm করুন। Manage Data-তে সরাসরি key fields batch edit করবেন না।

    9. Common mistakes¶

    Mistake Correct action
    Scheduled দেখে complete মনে করা। Scheduled মানে arrange হয়েছে। Completed মানে complete।
    Balance Qty (kg) > 0 দেখলেই Create Balance Requirement? = Yes select করা। শুধু remaining quantity সত্যিই dyeing করতে হলে select করুন।
    Balance Requirement Created? manually edit করা। এটি system duplicate-prevention field। manually edit করবেন না।
    Execution Pool-এ source order, yarn বা color edit করা। এগুলো upstream requirement, order বা item data থেকে আসা উচিত। আগে source form check করুন।
    Manage Data দিয়ে key fields batch edit করা। একটি record খুলে normalভাবে edit ও save করুন।

    10. Employee memory rule¶

    প্রথমে Status দেখুন, তারপর Source দেখুন, তারপর Yarn দেখুন, তারপর Color দেখুন, তারপর Qty দেখুন, শেষে Balance দেখুন।

    সবচেয়ে important কথা:

    Yarn Dyeing Execution Pool হলো Yarn Dyeing Work Card arrange করা, Execution Status follow-up করা, execution quantity control করা এবং Balance manage করার dyeing execution task pool। এটি নতুন dyeing requirement manually লেখার জায়গা নয়।
