    Customer Sample Analysis ব্যবহার নির্দেশিকা — বাংলা¶

    1. Form-এর উদ্দেশ্য¶

    Customer Sample Analysis হলো Production Department-এর customer elastic sample technical analysis form।

    এই form-এ record করা হয়:

    • sample structure, weaving method, width, thickness, handfeel, elongation এবং weight;
    • material composition এবং Sample Composition Breakdown;
    • BOM material, ends, consumption, dyeing requirement, dyeing process এবং color information;
    • customer sample image, analysis photo বা reference attachment।

    Record add করার পরে automation analysis result existing Elastic Item Master-এর OS technical data এবং sample composition information-এ write back করে।

    এই form sales order form নয়। ব্যবসায়িক/নিষ্পত্তি সংক্রান্ত তথ্য বা production schedule এই form-এ লিখবেন না।

    2. কারা এবং কখন ব্যবহার করবেন¶

    Main users: Production Department।

    আলাদা Design Department নেই। Production users including designers একই account এবং permission structure ব্যবহার করে।

    ব্যবহার করুন যখন:

    1. customer physical sample analysis-এর জন্য দেয়;
    2. customer photo বা existing Elastic information দেয় এবং technical information complete করতে হয়;
    3. Production users customer sample-এর material composition এবং OS BOM তৈরি করতে চায়;
    4. sample analysis result existing Elastic Item Master-এ write back করতে হয়।

    3. বর্তমান process¶

    • এই form-এ বর্তমানে workflow নেই।
    • এই form automation assistant দিয়ে existing Elastic Item Master update করে।
    • এই form শুধু existing record-এ write back করে। নতুন Elastic Item Master তৈরি করে না।

    4. Basic Order Information¶

    এই field-গুলো analysis কোন customer এবং কোন Elastic product master data-এর সাথে যুক্ত তা identify করে।

    Field Employee guidance
    Select Elastic সঠিক existing Elastic / Elastic Item Master record নির্বাচন করুন।
    Customer Code Elastic Item Master write back-এর key matching field; random change করবেন না।
    Elastic Code Elastic Item Master write back-এর key matching field; random change করবেন না।
    Country Customer country বা region; selection-এর পরে check করুন।
    Responsible Sales Traceability-এর জন্য responsible salesperson।
    Description Sample বা existing data অনুযায়ী product description।
    Elastic Type System-এ current option থেকে নির্বাচন করুন।
    Sales Account Ownership এবং visibility related field; ভুল selection data ownership প্রভাবিত করতে পারে।

    Operating rules:

    1. আগে সঠিক Select Elastic নির্বাচন করুন।
    2. Save করার আগে Customer Code এবং Elastic Code, target Elastic Item Master-এর সাথে match করে কিনা check করুন।
    3. Customer Code বা Elastic Code unclear হলে stop করুন এবং supervisor বা Jodoo Admin-কে জিজ্ঞাসা করুন।

    5. Basic Analysis Information¶

    এই field-গুলো sample-এর technical analysis result record করে।

    Field Employee guidance
    Analysis No. Traceability-এর জন্য analysis number।
    Accountable Designer Analysis-এর জন্য responsible production/design user।
    Attachment Customer sample image, analysis photo বা reference material upload করুন।
    Elastic Category System-এ current option থেকে নির্বাচন করুন।
    Structure Actual sample structure অনুযায়ী লিখুন।
    Weaving Method Actual weaving method অনুযায়ী লিখুন।
    Woven Pattern Woven pattern বা structure note।
    Knitted Pattern Knitted pattern বা structure note।
    Elastic Name Sample বা product name।
    Elastic Letter Sample letter বা marker।
    Handfeel Handfeel description।
    Greige Width / Greige Width Unit Greige width এবং unit।
    Finish Width / Finish Width Unit Finished width এবং unit।
    Thickness (mm) Thickness in mm।
    Weft Density (pick/cm) Weft density in pick/cm।
    Greige Elongation Greige elongation।
    Finish Elongation Finished elongation।
    Sample Weight (g/m) প্রতি meter sample weight।
    Created At Creation time বা analysis record time।

    Guidance:

    • Actual sample analysis অনুযায়ী field পূরণ করুন। Uncertain data guess করবেন না।
    • Width, thickness, elongation এবং weight যতটা সম্ভব accurate রাখুন।
    • Attachment analysis conclusion support করা উচিত।

    6. Customer Sample Material Analysis¶

    Subform name: Sample Composition Breakdown।

    এক row একটি material বা material position বোঝায়। এক row-তে একাধিক material মেশাবেন না।

    Field Employee guidance
    Position Material position বা structural position।
    Select Material সঠিক material record নির্বাচন করুন।
    Item Code অবশ্যই সঠিক হতে হবে; unsure হলে supervisor-কে জিজ্ঞাসা করুন।
    Item Category System data বা linked data থেকে confirm করুন।
    Material Type Material type।
    Standard Item Name Standard material name।
    材料中文名 System material Chinese name; exact system label preserve করুন।
    Base Unit Base unit।
    Brand Brand।
    No. of Ends Number of ends।
    Unit Consumption (g/m) প্রতি meter consumption; BOM এবং later usage judgment প্রভাবিত করে।
    Need Dyeing? Material dyeing দরকার কিনা decide করুন।
    Dyeing Process Dyeing দরকার হলে fill করুন।
    Color Name Color name।
    Target Color Ref. Target color reference।
    Quality Reference Type Quality reference type।
    Quality Reference No. Quality reference number।
    Remarks Notes।
    Confirm Status Row confirmed কিনা mark করে; unconfirmed row final BOM হিসেবে ব্যবহার করবেন না।
    单行BOM信息打包 System packing field। Manually edit করবেন না।

    7. Analysis record তৈরি¶

    1. Customer Sample Analysis খুলুন।
    2. New ক্লিক করুন।
    3. Select Elastic-এ সঠিক existing Elastic record নির্বাচন করুন।
    4. Customer Code এবং Elastic Code check করুন।
    5. Country, Responsible Sales, Description, Elastic Type এবং Sales Account check করুন।
    6. Analysis No., Accountable Designer এবং Attachment পূরণ করুন।
    7. Actual sample অনুযায়ী structure, weaving method, pattern, name, handfeel, width, thickness, elongation এবং weight পূরণ করুন।
    8. Sample Composition Breakdown-এ material composition row by row পূরণ করুন।
    9. Item Code, No. of Ends, Unit Consumption (g/m), Need Dyeing?, Dyeing Process, Color Name এবং Target Color Ref. check করুন।
    10. Confirmed material row-এর জন্য সঠিক Confirm Status set করুন।
    11. Save করার আগে Customer Code, Elastic Code এবং BOM information আবার check করুন।

    8. Save করার পরে কী হবে¶

    Confirmed automation assistant:

    • Automation name: 材料分析表填写 写回 总表单
    • Trigger form: Customer Sample Analysis
    • Trigger condition: new record Add record
    • Action: existing Elastic Item Master update
    • Matching conditions:
    • Elastic Item Master.Customer Code = Customer Sample Analysis.Customer Code
    • Elastic Item Master.Elastic Code = Customer Sample Analysis.Elastic Code

    Important rules:

    • Existing Elastic Item Master match না হলে system automatically নতুন record create করবে না।
    • Customer Sample Analysis শুধু existing Elastic Item Master record-এ write back করে।
    • তাই Customer Code এবং Elastic Code accurate হওয়া জরুরি।

    9. Elastic Item Master-এর সাথে সম্পর্ক¶

    Customer Sample Analysis, Elastic Item Master-এর OS technical data এবং sample material composition-এর important source।

    Confirmed OS technical field writeback includes, but is not limited to:

    • Ref. Analysis No. = Analysis No.
    • Accountable Designer = Accountable Designer
    • Attachment (OS) = Attachment
    • Elastic Category (OS) = Elastic Category
    • Structure (OS) = Structure
    • Weaving Method (OS) = Weaving Method
    • Woven Pattern (OS) = Woven Pattern
    • Knitted Pattern (OS) = Knitted Pattern
    • Elastic Name (OS) = Elastic Name
    • Handfeel (OS) = Handfeel
    • Greige Width (OS) = Greige Width
    • Greige Width Unit (OS) = Greige Width Unit
    • Finish Width (OS) = Finish Width
    • Finish Width Unit (OS) = Finish Width Unit
    • Thickness (mm) (OS) = Thickness (mm)
    • Weft Density (pick/cm) (OS) = Weft Density (pick/cm)
    • Greige Elongation (OS) = Greige Elongation
    • Finish Elongation (OS) = Finish Elongation
    • Sample Weight (g/m) (OS) = Sample Weight (g/m)
    • Created At (OS) = Created At
    • 全局BOM字符串集合 = 全局BOM字符串集合, field থাকলে

    10. সাধারণ ভুল¶

    • ভুল Select Elastic নির্বাচন করা।
    • Customer Code বা Elastic Code target Elastic Item Master-এর সাথে match না করা।
    • Uncertain technical data guess করা।
    • এক row-তে একাধিক material মেশানো।
    • ভুল Item Code নির্বাচন করা।
    • Unit Consumption (g/m)-এর বদলে total quantity লেখা।
    • Need Dyeing? ভুল judge করা।
    • Dyeing দরকার হলে Dyeing Process, Color Name বা Target Color Ref. খালি রাখা।
    • Unconfirmed Confirm Status row final BOM হিসেবে ব্যবহার করা।
    • Attachment upload না করা, ফলে sample source trace করা কঠিন হয়।

    11. Out of scope¶

    এই form ব্যবহার করা হয় না:

    • sales order;
    • ব্যবসায়িক/নিষ্পত্তি সংক্রান্ত তথ্য;
    • production scheduling;
    • নতুন Elastic Item Master তৈরি করতে।
