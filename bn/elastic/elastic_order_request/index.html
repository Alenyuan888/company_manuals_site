    Elastic Order Request ব্যবহার নির্দেশিকা — বাংলা¶

    1. Form-এর উদ্দেশ্য¶

    Elastic Order Request হলো Elastic sales order এবং sample request entry form।

    Sales users এই form-এ record করে:

    • customer এবং order information;
    • Job Type, Billing Type, customer PO এবং expected delivery date;
    • Elastic Order Details-এ product details, quantity, color, width এবং production routing;
    • attachment এবং remarks।

    Production users এই form production arrangement view থেকে দেখে। তারা mainly approval result, expected delivery date, product details এবং Production Arrangement Status check করে order pending, scheduled, completed না cancelled তা judge করে।

    এই form Elastic Sales Order helper table-এ sync করার source, এবং Sample request approve হলে Elastic Sample Development Work Order create করার source।

    2. কারা এবং কখন ব্যবহার করবেন¶

    Main users:

    • Sales
    • Production
    • Management

    আলাদা Design Department নেই। Designer involve হলে তাকে production users including designers হিসেবে ধরতে হবে, Production account এবং permission structure-এর অধীনে।

    ব্যবহার করুন যখন:

    1. Sales customer sample request বা bulk order পায়।
    2. Sales customer, order, product details, quantity, delivery date এবং attachment enter করতে চায়।
    3. Supervisor workflow-তে order approve করতে হবে।
    4. Job Type = Sample এবং supervisor approval হলে sample development work order create হওয়া দরকার।
    5. Production order-এর production arrangement status track করতে চায়।

    3. বর্তমান workflow¶

    Current workflow:

    1. Enter Order Details
    2. Supervisor Approval
    3. End

    Supervisor Approval একটি Regular Approval node। Daily users-এর জন্য মূল কথা: order information complete করুন, supervisor approval submit করুন, approval-এর পরে downstream handling continue করুন।

    4. Production Arrangement Status¶

    Production Arrangement Status Production-কে production planning বা arrangement view থেকে order status বুঝতে সাহায্য করে।

    Option Meaning
    Pending Order আছে, কিন্তু production এখনো arrange বা confirm হয়নি।
    Scheduled Production arrange হয়েছে, অথবা sample/bulk follow-up handling-এ ঢুকেছে।
    Completed Production-related handling complete।
    Cancelled Order cancel বা continue হবে না।

    Production users সাধারণত Approval Result = Approve এবং Production Arrangement Status = Pending record আগে দেখবে।

    5. Sales entry steps¶

    1. Elastic Order Request খুলুন।
    2. Select Customer-এ সঠিক customer নির্বাচন করুন।
    3. Customer Code, Company Name, Country, Responsible Sales এবং Sales Account check করুন।
    4. Current system options থেকে সঠিক Job Type নির্বাচন করুন, যেমন Sample বা Bulk।
    5. Current system options থেকে সঠিক Billing Type নির্বাচন করুন, যেমন FOC বা CHG।
    6. Customer PO No., Purchase Order Date এবং Expected Delivery Date পূরণ করুন।
    7. Attachment-এ customer order file, image, specification drawing বা reference upload করুন।
    8. Elastic Order Details-এ product line পূরণ করুন।
    9. Different style, color, width বা customer requirement হলে আলাদা detail line ব্যবহার করুন।
    10. Production Routing careful selection করুন, কারণ এটি sample বা bulk handling প্রভাবিত করে।
    11. Supervisor approval submit করুন।

    6. Sales Order Information¶

    এই field-গুলো customer, order source, sales ownership এবং production arrangement status identify করে।

    Field Employee guidance
    Select Customer সঠিক customer record নির্বাচন করুন।
    Customer Code Customer code; later sync এবং lookup-এ ব্যবহার হয়।
    Company Name Customer company name; customer select করার পরে check করুন।
    Country Customer country বা region।
    BIN Business Identification Number, applicable হলে।
    Responsible Sales Responsible salesperson।
    Sales Account Ownership এবং permission related field; অবশ্যই সঠিক হতে হবে।
    Job Type Sample / Bulk এবং অন্য work type আলাদা করে; current system options ব্যবহার করুন।
    Billing Type FOC / CHG এবং অন্য billing type আলাদা করে; current system options ব্যবহার করুন।
    Customer PO No. Customer PO বা reference number।
    Purchase Order Date Customer order date।
    Attachment Customer order, image, specification drawing বা reference material।
    Expected Delivery Date Expected delivery date; Production এটি careful check করে।
    Remarks Order remarks।
    Elastic Order No. System order number বা internal order number; lookup এবং sync-এ ব্যবহার হয়।
    Production Arrangement Status Production arrangement status।

    7. Elastic Order Details¶

    Elastic Order Details হলো product detail subform। এক row একটি Elastic style বা product detail বোঝায়।

    Field Employee guidance
    Select Elastic Related Elastic product record নির্বাচন করুন।
    Elastic Code Elastic product code; later sync এবং lookup-এ ব্যবহার হয়।
    Elastic Type System data বা current options থেকে confirm করুন।
    Description Product description।
    Elastic Color Color।
    Target Color Ref. Target color reference।
    Order Ref. Customer order-এর product reference।
    Width / Width Unit Width এবং unit।
    Net Weight Net weight, applicable হলে।
    Quantity / Quantity Unit Quantity এবং unit।
    Production Routing Production routing; Production এটি careful check করে।
    Remarks Product detail remarks।
    Product Picture Product image বা reference picture।
    Billing Type(辅助) System helper field, main-form Billing Type-এর সমান।
    Work Type(辅助) System helper field, main-form Job Type-এর সমান।

    Sales commercial বা settlement-related field company permission এবং process অনুযায়ী fill করতে হবে। Production users যদি production handling-এর জন্য দরকার নেই এমন commercial/settlement field দেখতে পায়, edit করবে না এবং Admin বা Alen-কে permission issue জানাবে।

    8. Production কীভাবে order দ্রুত filter করবে¶

    Production যদি Elastic Order No. না জানে, শুধু order number দিয়ে search করবেন না। এই order-এ filter করুন:

    1. Responsible Sales দিয়ে filter করুন।
    2. Job Type = Sample / Bulk দিয়ে filter করুন।
    3. Customer Code বা Company Name দিয়ে filter করুন।
    4. Production Arrangement Status দিয়ে filter করুন।
    5. Expected Delivery Date দিয়ে filter করুন।
    6. তারপর Elastic Code, Order Ref., Width বা Elastic Color দিয়ে specific product line খুঁজুন।

    Production-এর সবচেয়ে দরকারি field:

    • Approval Result
    • Production Arrangement Status
    • Job Type
    • Expected Delivery Date
    • Responsible Sales
    • Customer Code
    • Company Name
    • Elastic Order No.
    • Customer PO No.
    • Elastic Code
    • Elastic Color
    • Width
    • Quantity
    • Production Routing

    Recommended Production display fields: Approval Result, Production Arrangement Status, Job Type, Expected Delivery Date, Responsible Sales, Customer Code, Company Name, Elastic Order No., Customer PO No., Submission Time বা Purchase Order Date। এটি recommended configuration; system view already configured হয়েছে এমন confirmation নয়।

    9. Review & Approval¶

    Approval field supervisor-কে order continue করা যাবে কিনা decide করতে সাহায্য করে।

    Field Meaning
    Risk Assessment Risk assessment।
    PO Status PO status।
    Funding Status Funding status।
    L/C Readiness L/C readiness।
    Priority Priority।
    Remarks Approval remarks।
    Order Approval Order approval field।
    Approval Result Approval result।
    Approval Time Approval time।
    Approver Approver।
    Reason for Rejection Rejection reason।

    Order reject হলে Reason for Rejection check করুন এবং supervisor instruction follow করে continue করুন।

    10. Production Status¶

    Production status fields:

    • Sample Status
    • Bulk Production Status

    এই field-গুলো sample এবং bulk production follow-up handling track করে। Current system options ব্যবহার করুন।

    11. Save বা approval-এর পরে কী হবে¶

    Confirmed automation relationships:

    1. 松紧带销售订单更新到辅助表, Elastic Order Request data Elastic Sales Order helper table-এ sync করে।
    2. Job Type = Sample এবং Supervisor Approval approved হলে, 松紧带生产申请 批准后 生成松紧带样板Work Card, Elastic Sample Development Work Order create করে।

    Employee reminders:

    • Customer Code, Elastic Order No. এবং Elastic Code sync এবং matching প্রভাবিত করে।
    • ভুল Job Type sample work order create না হওয়া বা wrong downstream process ঘটাতে পারে।
    • ভুল Production Routing production/sample handling প্রভাবিত করে।

    12. সাধারণ ভুল¶

    • ভুল Select Customer নির্বাচন করা।
    • Sample এবং Bulk-এর মধ্যে ভুল Job Type নির্বাচন করা।
    • ভুল Billing Type নির্বাচন করা।
    • Expected Delivery Date খালি বা inaccurate রাখা।
    • এক Elastic Order Details row-তে different style, color বা width মেশানো।
    • Production Routing careful selection না করা।
    • Customer order, image বা specification attachment upload না করা।
    • Production শুধু order number দিয়ে search করে order খুঁজে না পাওয়া; combined filter ব্যবহার করুন।
    • Unapproved order confirmed production order হিসেবে handle করা।
