    Production Department — Elastic Order Request Q&A¶

    Applicable department¶

    Production Department

    Related form¶

    Elastic Order Request

    Before you start¶

    The Elastic Order Request records visible to Production Department are already approved orders.

    Production Department does not need to filter Approval Result = Approve during daily work.

    Production Department mainly needs to:

    • Check order information
    • Confirm whether production arrangement is needed
    • Update Production Arrangement Status

    Question 1: Which Display Fields should Production Department use for Elastic Order Request?¶

    Short answer¶

    The list fields should follow the production arrangement order:

    Status → Type → Delivery Date → Sales → Customer → Order No.

    This helps production staff quickly answer:

    • Does this order need action?
    • Is it Sample or Bulk?
    • When is the expected delivery date?
    • Which sales person is responsible?
    • Which customer is it?
    • Which order is it?

    Recommended Display Fields order¶

    The most important first 6 fields are:

    1. Production Arrangement Status
    2. Job Type
    3. Expected Delivery Date
    4. Responsible Sales
    5. Customer Code
    6. Company Name

    Field meaning:

    • Production Arrangement Status: check whether the order is pending, scheduled, completed, or cancelled.
    • Job Type: check whether it is Sample or Bulk.
    • Expected Delivery Date: check delivery priority.
    • Responsible Sales: know which sales person to contact for this order.
    • Customer Code: find the customer quickly.
    • Company Name: confirm the customer name.

    Fields that can continue after that:

    1. Elastic Order No.
    2. Customer PO No.
    3. Purchase Order Date
    4. Country
    5. Sample Status
    6. Bulk Production Status
    7. Workflow Status

    These fields help confirm order source, customer PO, order date, and later status.

    How to use Production Arrangement Status¶

    Production Department mainly updates this field:

    Production Arrangement Status

    Status meaning:

    • Pending: not arranged yet. Production needs to handle it.
    • Scheduled: production for sample or bulk has been arranged.
    • Completed: production-related handling is completed.
    • Cancelled: the order is cancelled or no longer needs handling.

    The most important daily work is to handle:

    Production Arrangement Status = Pending

    After opening an order, focus on Elastic Order Details¶

    After opening an order, Production Department should mainly check product details:

    • Elastic Code
    • Elastic Type
    • Description
    • Elastic Color
    • Target Color Ref.
    • Order Ref.
    • Width
    • Width Unit
    • Net Weight
    • Quantity
    • Quantity Unit
    • Production Routing
    • Remarks
    • Product Picture

    These fields help Production Department understand:

    • Which item should be made?
    • What color is needed?
    • What width is needed?
    • What quantity is needed?
    • Which production routing should be used?
    • Is there any picture or special remark?

    Question 2: If Production Department does not know the Elastic Order No., how can it quickly find the sales order information?¶

    Short answer¶

    If you do not know Elastic Order No., do not start by searching the order number.

    Use this filter order:

    Production Arrangement Status → Job Type → Responsible Sales → Customer Code / Company Name → Expected Delivery Date → Elastic Code / Color / Width / Order Ref.

    Step 1: Filter production arrangement status first¶

    Check this first every day:

    Production Arrangement Status = Pending

    This means these orders have not been arranged yet and need Production Department action.

    To view already arranged orders:

    Production Arrangement Status = Scheduled

    To find completed orders:

    Production Arrangement Status = Completed

    Step 2: Separate Sample and Bulk¶

    Check the field:

    Job Type

    For sample orders:

    Job Type = Sample

    For bulk orders:

    Job Type = Bulk

    Keep sample and bulk orders separate so the data is easier to read.

    Step 3: Filter by sales person¶

    If a sales person asks about an order but does not know the order number, filter first by:

    Responsible Sales = the sales person

    Example:

    Responsible Sales = Pete

    This narrows the order list to orders handled by that sales person.

    Step 4: Filter by customer¶

    Continue filtering by:

    Customer Code

    or:

    Company Name

    If you know the customer code, use Customer Code first.

    If you do not know the customer code, search by Company Name.

    Step 5: Filter by delivery date¶

    Production scheduling mainly depends on:

    Expected Delivery Date

    You can filter by:

    • Today
    • This week
    • Next 7 days
    • Next 30 days
    • Specific date

    Handle orders with the nearest delivery date first.

    Step 6: Use product information to find the exact item¶

    If the same customer still has many orders, use product information:

    • Elastic Code
    • Elastic Color
    • Width
    • Order Ref.
    • Customer PO No.
    • Production Routing

    Common filter combinations:

    • Pending + Sample + Responsible Sales + Customer Code
    • Pending + Bulk + Customer Code + Expected Delivery Date
    • Customer Code + Width + Elastic Color
    • Responsible Sales + Customer Code + Order Ref.

    Daily operation flow¶

    Production Department can work like this every day:

    1. Open Elastic Order Request.
    2. Filter Production Arrangement Status = Pending.
    3. Filter Job Type = Sample or Job Type = Bulk.
    4. Sort by Expected Delivery Date.
    5. Check Responsible Sales and Customer Code.
    6. Open the order and check Elastic Order Details.
    7. After arrangement is possible, change Production Arrangement Status to Scheduled.
    8. After completion, change it to Completed.
    9. For cancelled orders, change it to Cancelled.

    Simple memory rule¶

    Status → Job Type → Sales → Customer → Delivery Date → Product Details

    When Production Department sees an order, ask 4 questions first¶

    1. Is the current status Pending / Scheduled / Completed / Cancelled?
    2. Is this Sample or Bulk?
    3. When is the expected delivery date?
    4. Which elastic item is it?

    Once these 4 questions are clear, Production Department can quickly decide how to handle the order.
