    Customer Sample Analysis User Guide — English¶

    1. Purpose¶

    Customer Sample Analysis is a technical analysis form used by Production Department for customer elastic samples.

    The form records:

    • sample structure, weaving method, width, thickness, handfeel, elongation and weight;
    • material composition and Sample Composition Breakdown;
    • BOM materials, ends, consumption, dyeing requirement, dyeing process and color information;
    • customer sample images, analysis photos or reference attachments.

    After the record is added, an automation writes the analysis result back to an existing Elastic Item Master OS technical data section and sample composition information.

    This form is not a sales order form. Do not use it for commercial/settlement data or production scheduling.

    2. Users and use cases¶

    Main users: Production Department.

    There is no separate Design Department. Production users including designers use the same account and permission structure.

    Use this form when:

    1. a customer provides a physical sample for analysis;
    2. a customer provides a photo or existing Elastic information that needs technical completion;
    3. Production users need to build material composition and OS BOM for a customer sample;
    4. the sample analysis result must be written back to an existing Elastic Item Master.

    3. Current process¶

    • This form currently has no workflow.
    • This form uses an automation assistant to update an existing Elastic Item Master.
    • This form only writes back to existing records. It does not create a new Elastic Item Master.

    4. Basic Order Information¶

    These fields identify the customer and Elastic product master data that this analysis belongs to.

    Field Employee guidance
    Select Elastic Select the correct existing Elastic / Elastic Item Master record.
    Customer Code Key matching field for writing back to Elastic Item Master; do not change randomly.
    Elastic Code Key matching field for writing back to Elastic Item Master; do not change randomly.
    Country Customer country or region; review after selection.
    Responsible Sales Responsible salesperson for traceability.
    Description Product description from the sample or existing data.
    Elastic Type Select from current system options.
    Sales Account Ownership and visibility related field; wrong selection can affect data ownership.

    Operating rules:

    1. Select the correct Select Elastic first.
    2. Before saving, confirm that Customer Code and Elastic Code match the target Elastic Item Master.
    3. If Customer Code or Elastic Code is unclear, stop and ask a supervisor or Jodoo Admin.

    5. Basic Analysis Information¶

    These fields record the technical analysis result of the sample.

    Field Employee guidance
    Analysis No. Analysis number for traceability.
    Accountable Designer Production/design user responsible for the analysis.
    Attachment Upload customer sample images, analysis photos or reference materials.
    Elastic Category Select from current system options.
    Structure Enter according to the actual sample structure.
    Weaving Method Enter according to the actual weaving method.
    Woven Pattern Woven pattern or structural notes.
    Knitted Pattern Knitted pattern or structural notes.
    Elastic Name Sample or product name.
    Elastic Letter Sample letter or marker.
    Handfeel Handfeel description.
    Greige Width / Greige Width Unit Greige width and unit.
    Finish Width / Finish Width Unit Finished width and unit.
    Thickness (mm) Thickness in mm.
    Weft Density (pick/cm) Weft density in pick/cm.
    Greige Elongation Greige elongation.
    Finish Elongation Finished elongation.
    Sample Weight (g/m) Sample weight per meter.
    Created At Creation time or analysis record time.

    Guidance:

    • Fill fields based on actual sample analysis. Do not guess uncertain data.
    • Width, thickness, elongation and weight should be as accurate as practical.
    • Attachment should support the analysis conclusion.

    6. Customer Sample Material Analysis¶

    Subform name: Sample Composition Breakdown.

    One row represents one material or one material position. Do not mix several materials in one row.

    Field Employee guidance
    Position Material position or structural position.
    Select Material Select the correct material record.
    Item Code Must be correct; ask a supervisor if unsure.
    Item Category Confirm from system data or linked data.
    Material Type Material type.
    Standard Item Name Standard material name.
    材料中文名 System material Chinese name; preserve exact system label.
    Base Unit Base unit.
    Brand Brand.
    No. of Ends Number of ends.
    Unit Consumption (g/m) Consumption per meter; affects BOM and later usage judgment.
    Need Dyeing? Decide whether the material needs dyeing.
    Dyeing Process Fill when dyeing is needed.
    Color Name Color name.
    Target Color Ref. Target color reference.
    Quality Reference Type Quality reference type.
    Quality Reference No. Quality reference number.
    Remarks Notes.
    Confirm Status Marks whether the row is confirmed; unconfirmed rows must not be treated as final BOM.
    单行BOM信息打包 System packing field. Do not edit manually.

    7. Create an analysis record¶

    1. Open Customer Sample Analysis.
    2. Click New.
    3. Select the correct existing Elastic record in Select Elastic.
    4. Check Customer Code and Elastic Code.
    5. Check Country, Responsible Sales, Description, Elastic Type and Sales Account.
    6. Fill Analysis No., Accountable Designer and Attachment.
    7. Fill structure, weaving method, pattern, name, handfeel, width, thickness, elongation and weight based on the actual sample.
    8. Fill material composition row by row in Sample Composition Breakdown.
    9. Check Item Code, No. of Ends, Unit Consumption (g/m), Need Dyeing?, Dyeing Process, Color Name and Target Color Ref.
    10. Set the correct Confirm Status for confirmed material rows.
    11. Before saving, recheck Customer Code, Elastic Code and BOM information.

    8. What happens after saving¶

    Confirmed automation assistant:

    • Automation name: 材料分析表填写 写回 总表单
    • Trigger form: Customer Sample Analysis
    • Trigger condition: new record Add record
    • Action: update existing Elastic Item Master
    • Matching conditions:
    • Elastic Item Master.Customer Code = Customer Sample Analysis.Customer Code
    • Elastic Item Master.Elastic Code = Customer Sample Analysis.Elastic Code

    Important rules:

    • If no existing Elastic Item Master is matched, the system must not automatically create one.
    • Customer Sample Analysis only writes back to existing Elastic Item Master records.
    • Therefore Customer Code and Elastic Code must be accurate.

    9. Relationship with Elastic Item Master¶

    Customer Sample Analysis is an important source for OS technical data and sample material composition in Elastic Item Master.

    Confirmed OS technical field writeback includes, but is not limited to:

    • Ref. Analysis No. = Analysis No.
    • Accountable Designer = Accountable Designer
    • Attachment (OS) = Attachment
    • Elastic Category (OS) = Elastic Category
    • Structure (OS) = Structure
    • Weaving Method (OS) = Weaving Method
    • Woven Pattern (OS) = Woven Pattern
    • Knitted Pattern (OS) = Knitted Pattern
    • Elastic Name (OS) = Elastic Name
    • Handfeel (OS) = Handfeel
    • Greige Width (OS) = Greige Width
    • Greige Width Unit (OS) = Greige Width Unit
    • Finish Width (OS) = Finish Width
    • Finish Width Unit (OS) = Finish Width Unit
    • Thickness (mm) (OS) = Thickness (mm)
    • Weft Density (pick/cm) (OS) = Weft Density (pick/cm)
    • Greige Elongation (OS) = Greige Elongation
    • Finish Elongation (OS) = Finish Elongation
    • Sample Weight (g/m) (OS) = Sample Weight (g/m)
    • Created At (OS) = Created At
    • 全局BOM字符串集合 = 全局BOM字符串集合, if the field exists

    10. Common mistakes¶

    • Selecting the wrong Select Elastic.
    • Customer Code or Elastic Code does not match the target Elastic Item Master.
    • Guessing uncertain technical data.
    • Mixing several materials in one row.
    • Selecting the wrong Item Code.
    • Entering total quantity instead of Unit Consumption (g/m).
    • Judging Need Dyeing? incorrectly.
    • Leaving Dyeing Process, Color Name or Target Color Ref. empty when dyeing is needed.
    • Treating unconfirmed Confirm Status rows as final BOM.
    • Forgetting Attachment, making later sample traceability difficult.

    11. Out of scope¶

    This form is not used for:

    • sales orders;
    • commercial or settlement data;
    • production scheduling;
    • creating a new Elastic Item Master.
