    Yarn Dyeing Execution Pool User Guide¶

    1. What is this form for?¶

    Yarn Dyeing Execution Pool is the yarn dyeing execution task pool.

    It is not the first dyeing requirement form, and it is not the production Work Card. It turns a confirmed Yarn Dyeing Order into an executable task that the dyeing supervisor can arrange and follow.

    Simple relationship:

    Form Meaning for employees
    Yarn Dyeing Requirement Pool Dyeing requirement
    Yarn Dyeing Order Dyeing order
    Yarn Dyeing Execution Pool Dyeing execution task
    Yarn Dyeing Work Card Actual dyeing production card

    The dyeing supervisor mainly checks the yarn, color, arranged kg, finish date, status, and remaining balance quantity.

    2. Form process relationship¶

    The normal process is:

    1. Yarn Dyeing Requirement Pool creates a dyeing requirement.
    2. The requirement goes to Yarn Dyeing Order.
    3. After Yarn Dyeing Order is confirmed, the system creates Yarn Dyeing Execution Pool.
    4. The dyeing supervisor arranges Yarn Dyeing Work Card from Yarn Dyeing Execution Pool.
    5. After Yarn Dyeing Work Card selects the Execution, Execution Status becomes Scheduled.
    6. After Yarn Dyeing Work Card is completed, Execution Status becomes Completed.

    Yarn Dyeing Execution Pool is the source of Yarn Dyeing Work Card. Do not create a Work Card casually without the related Execution.

    Note: Output Record records dyeing output. It is not the completion button for Yarn Dyeing Execution Pool. Do not assume that filling a daily or output record will automatically complete the Execution. Scheduled / Completed mainly follows the arrangement and completion of Yarn Dyeing Work Card.

    3. What should I check first?¶

    Step 1: Check status¶

    Check Execution Status first.

    Execution Status Meaning
    Pending Not arranged to Yarn Dyeing Work Card yet.
    Scheduled Already arranged to Yarn Dyeing Work Card, but not completed yet.
    Completed This execution task is completed.

    Scheduled means arranged. Completed means finished.

    Step 2: Check source¶

    Check these source fields:

    • Yarn Dyeing Execution No.
    • Yarn Dyeing Order No.
    • Yarn Dyeing Requirement No.
    • Elastic Source WO No.
    • Elastic Source Order No.
    • Elastic Code
    • Job Type
    • Source Type
    • Requirement Type

    These fields tell you which order, Elastic work order, job type, and source this execution comes from.

    Step 3: Check yarn and color¶

    Check:

    • Yarn Item Code
    • Standard Item Name
    • Brand
    • Target Color Ref.
    • Color Name
    • Dyeing Process
    • Light Source
    • Yarn Color Standard No.
    • Special Requirements

    Do not check only Color Name. If Target Color Ref. exists, check it together.

    Step 4: Check quantity¶

    Field Meaning
    BOM Calculated Qty (kg) Requirement quantity calculated from BOM.
    Yarn Dyeing Order Qty (kg) Total dyeing quantity arranged in Yarn Dyeing Order.
    Execution Planned Qty (kg) Planned quantity for this execution.
    Balance Qty (kg) Remaining quantity.

    Balance Qty (kg) is normally calculated from Yarn Dyeing Order Qty (kg) minus Execution Planned Qty (kg). It is a system-calculated field. Do not edit it manually.

    Step 5: Check date and priority¶

    Check:

    • Planned Start Date
    • Planned Finish Date
    • Priority
    • Complete Time

    These fields help the dyeing supervisor arrange priority and follow actual completion.

    4. Smart Assistant / automation¶

    Several actions are automated in this form.

    Automation 1: Confirmed Yarn Dyeing Order creates Execution¶

    After Yarn Dyeing Order is confirmed, the system creates Yarn Dyeing Execution Pool.

    The system carries source, yarn, color, process, and quantity information. Employees should not manually add Execution records just for speed.

    Automation 2: Work Card arrangement updates status¶

    After Yarn Dyeing Work Card selects the Execution, Execution Status becomes Scheduled.

    Automation 3: Work Card completion updates status¶

    After Yarn Dyeing Work Card is completed, Execution Status becomes Completed, and Complete Time is recorded.

    Automation 4: Balance requirement is created only when needed¶

    When Balance Qty (kg) > 0 and the remaining quantity truly needs to continue dyeing, select Create Balance Requirement? = Yes.

    The system creates the follow-up or replenishment requirement and sets Balance Requirement Created? to Yes to prevent duplicate creation.

    Balance Qty (kg) > 0 does not always mean Create Balance Requirement? = Yes. Select it only when the remaining quantity still needs dyeing.

    5. How to understand Requirement Type¶

    Requirement Type shows the type of dyeing requirement.

    Requirement Type Meaning
    New Order New order or first dyeing requirement.
    Replenishment Remaining or additional quantity arranged again.
    Rework Re-dyeing or rework because of an issue.

    Requirement Type usually comes from the source form or system. Do not change it casually.

    6. Which fields can be changed?¶

    The dyeing supervisor may update these fields based on the real arrangement:

    • Planned Start Date
    • Planned Finish Date
    • Priority
    • Execution Planned Qty (kg)
    • Create Balance Requirement?
    • Complete Time
    • Special Requirements / Remarks

    Confirm the actual production arrangement before editing. Execution Planned Qty (kg) and Create Balance Requirement? affect later quantity and balance handling.

    Execution Status should normally be written back automatically from Yarn Dyeing Work Card.

    7. Which fields should not be changed casually?¶

    Do not casually edit:

    • Yarn Dyeing Execution No.
    • Yarn Dyeing Order No.
    • Yarn Dyeing Requirement No.
    • Elastic Source WO No.
    • Elastic Source Order No.
    • Elastic Code
    • Job Type
    • Source Type
    • Requirement Type
    • Yarn Item Code
    • Standard Item Name
    • Target Color Ref.
    • Color Name
    • Dyeing Process
    • Yarn Dyeing Order Qty (kg)
    • Balance Qty (kg)
    • Balance Requirement Created?

    These fields usually come from upstream forms, item data, color data, system calculation, or automation control. Wrong edits can make Work Card, source order, quantity, and balance handling inconsistent.

    8. Important fields not to batch edit in Manage Data¶

    Do not batch edit these fields in Manage Data:

    • Execution Status
    • Create Balance Requirement?
    • Balance Requirement Created?
    • Execution Planned Qty (kg)

    Backend batch editing may not trigger Smart Assistant / automation.

    Correct method:

    1. Open one Yarn Dyeing Execution Pool record.
    2. Edit the needed fields on the normal page.
    3. Save the record.
    4. Let the system trigger Smart Assistant / automation through the normal process.

    If many records need handling, confirm the method with the responsible person first. Do not directly batch edit key fields in Manage Data.

    9. Common mistakes¶

    Mistake Correct action
    Thinking Scheduled means completed. Scheduled means arranged. Completed means completed.
    Selecting Create Balance Requirement? = Yes whenever Balance Qty (kg) > 0. Select it only when the remaining quantity really needs further dyeing.
    Manually editing Balance Requirement Created?. This is a system duplicate-prevention field. Do not edit it manually.
    Editing source order, yarn, or color in Execution Pool. These should come from upstream requirement, order, or item data. Check the source form first.
    Batch editing key fields in Manage Data. Open one record, edit normally, and save.

    10. Employee memory rule¶

    Check status first, then source, then yarn, then color, then quantity, and finally balance.

    The most important sentence:

    Yarn Dyeing Execution Pool is the execution task pool for arranging Yarn Dyeing Work Card, following Execution Status, controlling execution quantity, and managing Balance. It is not the place to re-enter a dyeing requirement.
