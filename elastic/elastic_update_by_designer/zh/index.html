    Elastic Update by Designer 使用说明¶

    中文名称：设计部 Elastic 产品资料收集/补充表

    1. 表单用途¶

    Elastic Update by Designer 是 workflow 表单。Designer Department 在这里收集或补充特定客户、特定 Sales Account 的 Elastic 产品资料和 Common BOM Details。

    Sales Department 确认后，当前已启用的智能助手会在 Elastic Item Master 新增记录。本表当前不是直接修改已有 Elastic Item Master 的工具。

    2. 使用人员¶

    • Designer Department：选择客户和 Sales Account，填写产品资料和 Common BOM。
    • Sales Department：检查全部资料，选择 Confirm 或 Reject。
    • Jodoo Admin：处理 workflow、权限、公式或自动化异常。

    3. 当前流程¶

    1. Designer Editing
    2. Sales Review & Confirmation
    3. End

    Designer Editing¶

    Designer Department 负责客户、Sales Account、产品规格和 Common BOM，不应编辑 Sales Confirmation、Reject Reason 或 Approval Time。

    Sales Review & Confirmation¶

    • 资料正确完整：Sales Confirmation 选择 Confirm。
    • 资料错误或不完整：选择 Reject，并填写 Reject Reason。

    4. 主要字段¶

    • Select Elastic Customer
    • Customer Code；Company Name；Responsible Sales；Country(Region)
    • Sales Account；Serial No.
    • Elastic Type；Description；Elastic Color
    • Width；Width Unit；Net Weight (g/m)；Remarks
    • 全局BOM字符串集合
    • Sales Confirmation；Reject Reason；Approval Time
    • Common BOM Details

    字段的精确 Jodoo 组件类型、必填设置和关联带出映射如未确认，均为 pending_verification。

    5. 显示与审核规则¶

    • Sales Confirmation = Confirm：显示 Approval Time。
    • Sales Confirmation = Reject：显示 Reject Reason。
    • Reject Reason 在 Reject 时必须填写。
    • Approval Time 的精确赋值方式为 pending_verification。

    6. Common BOM Details¶

    Designer Department 只填写一份 Common BOM。当前已确认的字段名称包括：

    • Select Data；Position；Item Code；Item Category；Material Type
    • Standard Item Name；材料中文名字；Brand；Base Unit
    • No. of Ends；Unit Consumption (g/m)
    • Need Dyeing?；Dyeing Process；Target Color Ref.；Color Name
    • Quality Reference Type；Quality Reference No.；Light Source
    • Remarks；Confirm Status；FBS单行BOM信息打包

    不清楚的 BOM 信息可以留空，但不能猜填。全局 BOM 和单行打包字段由系统使用，普通人员不要修改。

    7. Designer 操作步骤¶

    1. 打开 Elastic Update by Designer，点击 New。
    2. 在 Select Elastic Customer 选择客户。
    3. 检查 Customer Code、Company Name、Responsible Sales 和 Country(Region)。
    4. 选择正确的 Sales Account。
    5. 填写产品类型、描述、颜色、宽度、单位、净重和备注。
    6. 在 Common BOM Details 逐行填写已知材料资料。
    7. 检查材料编码、单位、根数、单耗、染色和质量参考。
    8. 不要修改审核字段或系统打包字段。
    9. 提交到 Sales Review & Confirmation。

    8. Sales 审核步骤¶

    1. 检查客户和 Sales Account。
    2. 检查产品规格和单位。
    3. 检查每一行 Common BOM。
    4. 正确完整时选择 Confirm。
    5. 错误或不完整时选择 Reject，并写清 Reject Reason。

    9. Confirm / Reject 后的结果¶

    Confirm¶

    当前智能助手已启用，触发来源为 Update Record，当前已确认条件为：

    • Sales Confirmation = Confirm
    • Sales Account is not empty

    符合条件后执行 Add Record to Elastic Item Master，并把 Common BOM Details 复制到：

    • Sample Composition Breakdown (OS)
    • Sample Composition Breakdown (FBS)

    Reject¶

    • 不触发 Elastic Item Master 同步。
    • 必须填写 Reject Reason。

    10. 常见错误¶

    • 没有选择 Select Elastic Customer。
    • Sales Account 为空或选错。
    • Width、Net Weight 或 Unit Consumption 单位错误。
    • 不清楚的 BOM 信息被猜填。
    • Designer 修改审核字段。
    • Sales 未检查 Sales Account 或 BOM 就 Confirm。
    • Reject 后没有填写 Reject Reason。
    • 误以为本表会直接修改已有 Elastic Item Master。

    11. pending_verification¶

    以下内容尚未完整核对，不得臆测：

    • 未确认字段的精确 Jodoo 组件类型。
    • Select Elastic Customer 的完整关联字段映射。
    • 全局BOM字符串集合和 FBS单行BOM信息打包的精确公式。
    • Approval Time 的精确赋值方式。
    • Common BOM 到 OS/FBS BOM 的逐字段精确映射。
